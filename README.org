* 这是什么
  - emacs补全插件，内部使用libclang。特点是直接hack emacs，使得传递buffer内容时用的emacs进程本身的内存，而不是临时文件(其他补全无一例外会使用临时文件)
* 功能
  - 补全c/c++代码
  - 配合flymake检查语法错误
* 注意
  - 只支持win32平台的emacs，并且版本要大于等于23.2
* 安装
  - 到[[http://www.pleee.com/lcEngine.html]]下载最新的bin，将libclang放到MinGW的bin目录，其它放emacs的bin目录
  - 下载并安装vs2008 sp1运行库
  - 将lcEngine.el添加到emacs的load-list目录中
  - 运行lcEngineConfig.exe一次, 这会将lcEngine.dll添加到emacs的导入表中
* 用法
  - 在.emacs:中添加：
#+BEGIN_EXAMPLE
(add-to-list 'load-path "where the lcEngine.el exists")
(require 'lcEngine)
(add-hook 'c-mode-common-hook
	  (lambda ()
	    (local-set-key (kbd "C-.") 'ac-complete-lcEngine)
             (ac-lcEngine-setup) ;should comment if testing
	    ))
#+END_EXAMPLE
  需要auto complete，yasnippet, flymake
  - 写一般的小程序都可以补全了，如果还想定制更多可以在cpp文件的同级目录里添加lcEngine.txt，内容如：
#+BEGIN_EXAMPLE
-D__MSVCRT__
-ID:\boost
#+END_EXAMPLE
* 卸载
  - 删除被patch过的emacs.exe更名emacs.bak.exe为emacs.exe, 删除lcEngine.el相关的配置
